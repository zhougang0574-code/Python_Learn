01 FastAPI 是什么
CORE FastAPI 与 Spring Boot 对比
▶

FastAPI ≈ Spring Boot，但更轻量。两者都是 Web 后端框架，核心概念一一对应。

概念Java Spring BootPython FastAPI
框架定位Web后端框架Web后端框架
路由注解@GetMapping("/path")@app.get("/path")
启动方式内嵌Tomcat内嵌Uvicorn
数据校验@Valid + Bean ValidationPydantic（自动）
文档生成需引入Swagger内置，自动生成
CORE 最小可运行示例
▶
python · main.py
from fastapi import FastAPI  # 相当于Spring Boot的@SpringBootApplication

app = FastAPI()  # 创建应用实例，相当于Spring容器

# @app.get 对应 @GetMapping
@app.get("/hello")
def say_hello():
    # 直接return字典，FastAPI自动序列化为JSON
    # 不需要@ResponseBody，默认就是
    return {"message": "Hello World"}
bash · 启动命令
# main:app => 文件名main.py 里的 app 变量
# --reload => 代码修改后自动重启（相当于Spring DevTools）
uvicorn main:app --reload
💡 自动文档
启动后访问 http://127.0.0.1:8000/docs 即可看到自动生成的 Swagger 文档，零配置。
02 Uvicorn
KEY Uvicorn 的作用
▶

FastAPI 本身不能直接处理 HTTP 请求，需要一个 HTTP 服务器来驱动它。Uvicorn 就是这个服务器，相当于 FastAPI 的内嵌 Tomcat。

角色Java 生态Python 生态
Web 框架Spring BootFastAPI
HTTP 服务器TomcatUvicorn
关系Spring Boot 内嵌 TomcatFastAPI 需配合 Uvicorn
⚠️ 区别
Spring Boot 把 Tomcat 打包进去了，你感觉不到它存在。FastAPI 没有内嵌，需要单独安装并显式调用 Uvicorn，所以安装时两个要一起装：pip install fastapi uvicorn
03 路径参数 vs 查询参数
CORE 路径参数（Path Parameter）
▶

写在 URL 路径里的参数。Java 用 @PathVariable，FastAPI 直接在路径字符串里用 {变量名}，函数参数同名即自动绑定。

python
# Java: @GetMapping("/user/{id}") + @PathVariable int id
@app.get("/user/{user_id}")
def get_user(user_id: int):  # 类型注解，FastAPI自动做类型转换和校验
    return {"user_id": user_id}
CORE 查询参数（Query Parameter）
▶

函数里有但路径里没有的参数，自动视为查询参数。有默认值 = 选填，无默认值 = 必填。

python
# 访问：/items?skip=0&limit=10
# Java: @RequestParam(defaultValue="0") int skip
@app.get("/items")
def get_items(skip: int = 0, limit: int = 10):
    # 有默认值 = 选填，无默认值 = 必填
    return {"skip": skip, "limit": limit}
📌 识别规则
路径中出现 {变量名} → 路径参数。函数参数有但路径里没有 → 查询参数。FastAPI 自动区分，无需额外注解。
04 请求体 Request Body
CORE Pydantic BaseModel
▶
概念Java Spring BootPython FastAPI
接收请求体@RequestBody + DTO类BaseModel + 函数参数
数据校验@Valid + @NotNull等Pydantic字段类型直接校验
请求方式POST / PUTPOST / PUT
python
from fastapi import FastAPI
from pydantic import BaseModel  # 相当于Java的DTO/VO类

app = FastAPI()

# 定义请求体结构，相当于Java里的：
# public class UserDTO {
#     private String name;
#     private int age;
# }
class User(BaseModel):
    name: str
    age: int
    email: str = "无"  # 有默认值 = 选填

# 参数类型是BaseModel，FastAPI自动识别为请求体
# 不需要@RequestBody注解
@app.post("/user")
def create_user(user: User):
    return {"name": user.name, "age": user.age}
KEY 路径参数 + 请求体 混用
▶
python
# 路径里有 {item_id} → 路径参数
# 参数类型是BaseModel → 请求体
# FastAPI 自动区分，不会混淆
@app.put("/item/{item_id}")
def update_item(item_id: int, user: User):
    return {"item_id": item_id, "user": user}
💡 model_dump()
BaseModel 提供 model_dump() 方法将实例转为字典。子类继承 BaseModel 后自动拥有，类似 Java 基类的 toMap()。
05 响应模型 Response Model
CORE response_model — 过滤敏感字段
▶

Java 里用单独的 VO 类控制返回字段，FastAPI 用 response_model 参数。FastAPI 会自动过滤掉输出模型里没有的字段。

python
class UserIn(BaseModel):   # 入参，含敏感字段，相当于Java DTO
    name: str
    age: int
    password: str

class UserOut(BaseModel):  # 出参，不含敏感字段，相当于Java VO
    name: str
    age: int

# FastAPI 自动把password过滤掉，前端收不到
@app.post("/user", response_model=UserOut)
def create_user(user: UserIn):
    return user  # 虽然包含password，但response_model会过滤
EXTENDED response_model_exclude_unset
▶

只返回客户端实际传入的字段，没传的字段（即使有默认值）不返回。典型场景：PATCH 局部更新。

python · 示例对比
# 客户端只传了：{"name": "手机"}

# 不加 exclude_unset，返回：
# {"name": "手机", "price": 0.0, "stock": 0, "description": "暂无描述"}

# 加了 exclude_unset=True，返回：
# {"name": "手机"}

@app.post("/item", response_model=Item, response_model_exclude_unset=True)
def create_item(item: Item):
    return item
KEY 实际项目中入参出参命名约定
▶

实际项目中入参和出参几乎都是分开定义的，和 Java 拆分 DTO / VO / PO 是同一个道理。

类名用途
UserCreate创建时的入参（含password）
UserUpdate更新时的入参（字段可能都选填）
UserOut返回给前端的出参（过滤敏感字段）
UserInDB存数据库的内部模型（含hash_password等）
06 状态码与错误处理
CORE status_code — 指定成功状态码
▶

Java 用 ResponseEntity.status(201).build()，FastAPI 直接在装饰器里指定 status_code。

python
# 201 = Created，创建资源时常用
# 默认不写就是 200
@app.post("/user", status_code=201)
def create_user():
    return {"message": "创建成功"}
CORE HTTPException — 抛出错误
▶

Java 抛 ResponseStatusException，FastAPI 用 HTTPException，使用 raise 而不是 throw。

python
from fastapi import FastAPI, HTTPException

fake_db = {1: "张三", 2: "李四"}

@app.get("/user/{user_id}")
def get_user(user_id: int):
    # Java: throw new ResponseStatusException(HttpStatus.NOT_FOUND, "用户不存在")
    if user_id not in fake_db:
        raise HTTPException(status_code=404, detail="用户不存在")

    return {"user_id": user_id, "name": fake_db[user_id]}
