# CLAUDE.md

## 这是什么项目

用户是 Java 后端开发者，在这个项目里系统学习 **Python 基础**（不含 LangChain/LangGraph，那部分在另一个独立项目里学）。

完整的章节表、每章状态（⬜ 未开始 / 🔶 进行中 / ✅ 已完成）都记录在 **PROJECT.md**。每次开始工作前先读 PROJECT.md，确认当前进度，不要让用户重复说明。

## 项目约定

- 笔记放在 `notebooks/序号_主题/` 下，每章固定两个文件：
  - `知识点.ipynb` — 概念讲解 + 示例代码，必须完整覆盖该章知识点
  - `练习.ipynb` — 练习题，**每题固定两个 cell**：
    1. 一个 markdown cell，只写标题，格式 `### N、标题`（不要把题目描述塞进 markdown cell）
    2. 一个 code cell，题目描述/测试用例/期望输出全部以 `#` 注释的形式写在这个 code cell 顶部，空一行后跟 `# 在这里写你的代码`，下面留空——**题目正文必须在 code cell 里，不能写进 markdown cell**（写进 markdown 会被当成标题渲染，格式会乱）
    
    完整模板：
    ```
    [markdown cell] ### 1、标题

    [code cell]
    # 题目描述...
    # 测试用例：...
    # 期望输出：...

    # 在这里写你的代码
    ```
    不要直接把答案写满，照着已有的 `notebooks/01_环境与基础语法/练习.ipynb`、`notebooks/02_数据类型与容器/练习.ipynb` 的格式抄就不会错
- 章节分两类（PROJECT.md 表里的"类型"列）：
  - **复习/半复习**：用户已掌握的内容，知识点照样讲全，但练习题数量少（3~8 题）
  - **新学**：知识点 + 练习都要完整覆盖
- notebook 用中文 markdown 标题分节（如 `### 1、xxx`），风格参考已有的 `notebooks/01_环境与基础语法/` 和 `notebooks/07_文件操作/`

## 工作流程

1. 看 PROJECT.md 章节表，找到 🔶 进行中 或下一个 ⬜ 未开始的章节，从那里继续，不要跳章节、也不要一次把后面所有章节的 notebook 都建好
2. 先产出该章 `知识点.ipynb`，讲完知识点再产出 `练习.ipynb`
3. 用户做完练习、贴代码或要求批改时，检查正确性、指出 bug 和可优化点（参考已有风格：直接指出问题 + 给修正方式，不用展开成大段讲解）
4. 用户确认这一章学完后，把 PROJECT.md 里对应行的状态改成 ✅，再进入下一章
5. 讲解新概念时，用户有 Java 背景，可以适当对比 Java（如静态类型 vs 动态类型、`{}` vs 缩进），但不用每个点都强行类比

## 最终目标

学完 PROJECT.md 最后一章「类型注解 & 函数签名精讲」后，用户应该能看懂任意第三方库（包括之后要学的 LangChain/LangGraph）的函数/方法签名，知道参数怎么传。这是设计最后一章的直接动机，讲解时要呼应这个目标。
